Hexo博客搭建

1.安装Termux(如果你是PC用户，请跳转到第二个环节)
打开后会下载数据，请配合科学上网

忘记了非常重要的事情，部分命令需用权限运行
2020.4.30日补充

输入tsu并回车
没有安装的根据提示安装

2.注册Github(有的话请跳到命令处)
注册环节也可能会用到科学上网，并New一个仓库，仓库名 你的Github名字.github.io
3.配置环境&安装软件包

pkg install git #安装Git
pkg install nodejs-lts #配置nodejs环境
npm install hexo-cli -g #会有黄色的警告，可以忽略，没有影响
mkdir '文件名' #如果你是PC用户请创建在你想要的磁盘内，Termux可以直接使用该命令
cd '进入文件内'
hexo init #初始化Hexo
git init #初始化Git
git config --global user.name #你的github用户名
git config --global user.email #注册github的邮箱
pkg install openssh #安装openssh
ssh-keygen -t rsa -C "你注册Gayhub用的邮箱" 
不断回车，然后在/data/data/com.termux/files/home/.ssh/此路径，打开id_rsa.pub文件并复制内容到Github，点击你的头像找到SSH and GPG keys，新建SSH Key粘贴就行了
如果你是PC，找到你新建那个文件的目录就行了
ssh -T git@github.com #如果出现你的名字便添加SSH成功
npm install --save hexo-deployer-git #安装上传博客工具
出现报错的话，可以试下
pkg install coreutils
然后再使用前一个命令
或者换源
完成后
hexo s #本地启动博客，在浏览器中输入http://localhsot:4000 即可进入博客
如果出现端口占用
hexo s -p 5000 #换成5000端口
然后就可以本地预览博客了
想要关闭预览可以用 CTR + C 键退出


Hexo基础命令

hexo s #本地启动博客
hexo generate 简单点用 hexo g #渲染生成文件
hexo deploy 也可以用 hexo d #向Gayhub提交你的博客
hexo clean #清除缓存，清除前一次生成的渲染文件，在你需要提交博客时，请使用该命令
hexo new 名字 #为创建一个博客文件，注意，不要跟下面的搞混了!
hexo new '名字' #为创建一个文章，你可以在/source/_post/内找到它，并写文章上传
#注:Hexo的文章使用Markdown语言编辑


使用Sakura主题


cd ‘博客目录内’
git clone git@github.com:honjun/hexo-theme-sakura.git #克隆博客文件

注意阅读官方文档！阅读官方文档!读文档!
根据官方文档操作
而且要记得安装依赖


npm i

使用后，如果出现


(node:13578) [DEP0061] DeprecationWarning: fs.SyncWriteStream is deprecated.

可以忽略，没有影响


其他主题更换教程
把其他主题放在你放博客的文件/themes 中，并删除landscape主题，然后在博客根目录的_config.yml文件内，找到theme并修改为主题名(你放在themes文件内文件夹的名字)

hexo s #便可预览你换好的博客啦

3.添加RSS
cd 进入博客根目录


npm install hexo-generator-feed –save #安装RSS

然后在，博客根目录，记住是根目录的_config.yml，添加如下内容

feed: 
ptype: atom 
path: atom.xml 
limit: 20 
hub: 
content: 
content_limit: 140 
content_limit_delim: ' ' 
order_by: -date 
icon: icon.png

具体内容，请参照hexo-generator-feed

Hexo日常使用的一些问题

可能是我个人的问题，但可能会对你有所帮助
1.Termux报错


events.js:174

在博客目录内放进了文件，导致hexo s博客无法启动，原因是hexo没有权限打开博客目录内的文件，于是乎我按照其他正常文件的权限修改为777，结果还是一样，这个玩意非常迷，与Termux的文件权限问题有关(这里我并不知道怎么解释)，可是我明明授权了Terumx的Root权限了啊?难道是我的syslock没有解?当然也不是…总是上传再clone肯定不是办法，后来我又想出了一个办法，使用cp与mv命令把其他文件拷贝到指定目录中，后来就一直用了…

6月21日更新

因为我的Termux出了点问题，导致命令无法使用，于是我清除了数据，再次搭建博客时经过几小时无果

然后没有办法的我只能认为是miui系统问题导致的权限不完整，说干就干，没想到养老一年的我又走上了不归路，刷了Havoc-os，不得不说类原生系统开机画面是真好康!而且使用非常舒适!偏题了，咳咳…
没想到这问题就解决了，也许你不明白我在说什么，那么恭喜你没有遇到这个问题。


2.无法使用hexo命令


Usage: hexo < command >

这也是个非常让人头疼的问题
起先是根据某度上的回答，又搭了一遍，这一搭还差点把我博客搞没了，气死(主要是手残)
后来再研究，说是因为hexo s命令在某个版本被独立出来了，使用命令


npm install hexo-server –save #安装hexo服务
